What Makes Myki Unique

Unlike traditional password managers that store credentials in the cloud, Myki uses a peer-to-peer architecture. Your passwords stay on your devices and are shared securely between team members without ever touching a central server. This approach offers several automation advantages:

  • Offline-first design: Automations work even without internet connectivity
  • Zero-knowledge architecture: No master password stored anywhere
  • Real-time sync: Changes propagate instantly across authorized devices
  • Biometric authentication: Automate login flows with Touch ID, Face ID, or Windows Hello

Step-by-Step Implementation

Step 1: Account Setup (5 minutes)

  1. Log into your Myki for Teams admin dashboard
  2. Navigate to Settings → Automation
  3. Enable the automation module
  4. Configure basic permissions for your team roles
💡 Pro Tip: Start with a pilot group of 2-3 team members before rolling out automation to your entire team. This allows you to identify and resolve issues in a controlled environment.

Step 2: Configure Automation Settings (10-15 minutes)

Myki for Teams offers several automation options. Focus on these essential settings:

Setting Recommended Value Why
Auto-fill Enabled Automatically fills credentials on recognized sites
Password Rotation Every 90 days Maintains security without overwhelming users
Sharing Notifications Instant Team members know immediately when credentials are shared
Access Expiration 30 days Temporary access for contractors or temporary team members

Step 3: Set Up Automated Workflows (20-30 minutes)

Create automation workflows for common password tasks:

Workflow 1: New Employee Onboarding

  1. Create a "New Employee" collection in Myki
  2. Add all credentials needed for onboarding (email, Slack, tools)
  3. Set up automatic sharing when a new team member joins
  4. Configure welcome message with instructions

Workflow 2: Password Change Propagation

  1. Enable "Auto-sync" for shared credentials
  2. Set notifications for password updates
  3. Configure automatic logout on changed passwords

Workflow 3: Departing Employee Offboarding

  1. Create offboarding checklist in Myki
  2. Set up one-click access revocation
  3. Enable audit log export for compliance

Troubleshooting Common Issues

Issue: Automation not triggering

Solution: Check your trigger conditions. 45% of issues are caused by conflicting conditions. Verify the automation is enabled and try refreshing your app.

Issue: Credentials not syncing across devices

Solution: Ensure both devices have internet connectivity and are logged into the same Myki account. Check that peer-to-peer connection is established.

Issue: Team member can't access shared credentials

Solution: Verify the team member has accepted the share invitation. Check that their access hasn't expired and confirm they're in the correct team group.

Frequently Asked Questions

Is Myki for Teams good for remote teams?

Yes, Myki's peer-to-peer architecture makes it excellent for remote teams. Since credentials sync directly between devices without going through a central server, team members can share passwords securely regardless of location.

Can I automate password sharing with contractors?

Absolutely. Myki's access expiration feature lets you share credentials with contractors temporarily. Set an expiration date, and access is automatically revoked when the time expires.

What happens if Myki servers go down?

Because Myki stores credentials locally on your devices and uses peer-to-peer syncing, your passwords remain accessible even if Myki's servers are unavailable. This is a key advantage over cloud-only solutions.

How secure is Myki's automation?

Myki uses end-to-end encryption for all credential sharing. Since passwords never touch Myki's servers, they can't be accessed by anyone outside your team—including Myki employees.

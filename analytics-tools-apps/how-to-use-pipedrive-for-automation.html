How to Use Pipedrive for Automation: Complete Sales Workflow Guide

Pipedrive's automation features eliminate repetitive sales tasks, ensuring no lead falls through the cracks while saving hours each week. This guide shows you how to set up powerful automations that handle routine work while you focus on closing deals.

Pipedrive Automation Overview

Pipedrive offers native automation through its Automation feature, available on Advanced plans and above. These automations run automatically based on triggers you define.

What You Can Automate

  • Deal movements – Move deals between stages based on criteria
  • Activities – Create, assign, and schedule follow-ups automatically
  • Emails – Send automated email sequences
  • Field updates – Update deal, person, and organization fields
  • Assignments – Route deals to the right sales rep automatically
  • Notifications – Alert team members of important events
  • Webhooks – Send data to external systems

Automation Plans

Getting Started with Automations

Accessing the Automation Builder

  1. Log into Pipedrive
  2. Go to Settings → Automation
  3. Click "+ Add automation"
  4. Name your automation (e.g., "New Lead Follow-up")

Understanding Triggers and Actions

Every automation has two parts:

  • Trigger – What starts the automation (e.g., "Deal created")
  • Action – What happens automatically (e.g., "Create activity")

Available Triggers

  • Deal created
  • Deal stage changed
  • Deal updated (field value changed)
  • Activity added/completed
  • Person/Organization created
  • Lead added (for Lead Inbox)

Available Actions

  • Update deal, person, or organization field
  • Create activity
  • Move deal to stage
  • Send email
  • Add note
  • Send webhook
  • Create recurring activity

Deal Stage Automation

Auto-Move Deals Based on Criteria

Automatically progress deals through your pipeline:

Example: Move Deal After First Call

  1. Trigger: Activity completed where type = "Call" and deal stage = "New"
  2. Condition: Activity marked as "Done"
  3. Action: Move deal to "Qualified" stage

Example: Auto-Progress Based on Field

  1. Trigger: Deal field "Budget confirmed" changes to "Yes"
  2. Action: Move deal to "Discovery" stage
  3. Action: Create "Send proposal" activity due in 2 days

Setting Up Stage-Based Workflows

Create different automations for each pipeline stage:

Stage Trigger Automated Actions
New Deal created Create "First contact" activity, assign to round-robin
Qualified Deal moved to Qualified Schedule demo call, send qualifying email
Proposal Deal moved to Proposal Create "Follow up" activity in 3 days, send proposal template
Negotiation Deal moved to Negotiation Notify manager, schedule internal review
Won Deal marked Won Create handoff activity, send welcome email, update customer status

Deal Value-Based Routing

Route high-value deals differently:

  1. Trigger: Deal created with value > $50,000
  2. Action: Assign to senior sales rep
  3. Action: Add "High priority" label
  4. Action: Send notification to sales manager

Activity Automation

Creating Follow-Up Activities Automatically

Never miss a follow-up with automated activity creation:

Example: Post-Meeting Follow-up

  1. Trigger: Activity "Meeting" marked complete
  2. Action: Create "Send follow-up email" activity due in 1 day
  3. Action: Create "Schedule next meeting" activity due in 3 days

Recurring Activities

Automate regular check-ins:

  1. Trigger: Deal moved to "Customer" stage
  2. Action: Create recurring "Customer check-in" activity every 30 days

Activity Assignment

Route activities to the right person:

  1. Trigger: Deal created in "Technical Services" pipeline
  2. Action: Create "Technical review" activity assigned to engineering lead

Overdue Activity Alerts

While Pipedrive doesn't have native overdue triggers, you can use webhooks:

  1. Set up webhook automation to external service (Zapier, Make)
  2. Check for overdue activities on schedule
  3. Send Slack/Teams notification for overdue items

Email Automation

Pipedrive Email Features

Pipedrive's email automation (Professional plan+) includes:

  • Automated email sequences
  • Smart BCC to log emails automatically
  • Email templates with merge fields
  • Scheduled email sending

Setting Up Email Sequences

  1. Go to Email → Templates → Sequences
  2. Create a new sequence
  3. Add emails with timing (e.g., Day 1, Day 3, Day 7)
  4. Set enrollment criteria (who gets the sequence)
  5. Activate the sequence

Example: New Lead Nurture Sequence

  • Email 1 (Day 0): Welcome email with intro
  • Email 2 (Day 3): Value proposition
  • Email 3 (Day 7): Case study/social proof
  • Email 4 (Day 14): Demo offer

Trigger-Based Email Automation

  1. Trigger: Deal moved to "Demo scheduled"
  2. Action: Send confirmation email template
  3. Action: Add calendar invite link to note

Email Merge Fields

Personalize automated emails with merge fields:

  • {{person.name}} – Contact's name
  • {{person.first_name}} – First name
  • {{organization.name}} – Company name
  • {{deal.title}} – Deal title
  • {{owner.name}} – Sales rep's name

Lead Inbox Automation

Pipedrive's Lead Inbox captures leads from forms, chat, and imports. Automate the conversion process:

Auto-Convert Qualified Leads

  2. Action: Create person and organization

Lead Routing

Assign leads automatically based on criteria:

  1. Trigger: New lead from specific source
  2. Condition: If source = "Enterprise website"
  3. Action: Assign to enterprise sales team
  4. Action: Add "Enterprise" label

Web Form Integration

Connect web forms to Pipedrive:

  1. Use Pipedrive's web forms or integrate via API
  2. Map form fields to person/organization/deal fields
  3. Set up automation for new form submissions

Advanced Workflows

Multi-Step Automations

Chain multiple actions to a single trigger:

Example: New Enterprise Deal Workflow

  1. Trigger: Deal created with value > $100,000
  2. Action 1: Assign to enterprise sales rep
  3. Action 2: Add "Enterprise" label
  4. Action 3: Create "Discovery call" activity due in 1 day
  5. Action 4: Send Slack notification to sales channel
  6. Action 5: Add internal note with qualification questions

Conditional Logic

Add conditions to your automations:

  • Only run if: Deal value > X
  • Only run if: Pipeline = "Sales"
  • Only run if: Stage is "Proposal" or earlier
  • Only run if: Owner is specific person

Round-Robin Assignment

Distribute leads fairly among sales reps:

  1. Create a custom field "Assignment queue"
  2. Set up automation that cycles through owners
  3. Use Zapier or Make for true round-robin logic

Won Deal Handoff

  1. Trigger: Deal stage = "Won"
  2. Action: Update customer status field
  3. Action: Create "Onboarding" activity for customer success
  4. Action: Send welcome email sequence
  5. Action: Move to customer pipeline

Lost Deal Follow-up

  1. Trigger: Deal stage = "Lost"
  2. Action: Add to "Lost deals" list
  3. Action: Schedule "Re-engagement" activity in 90 days
  4. Action: Send feedback request email

Best Practices

Start with High-Impact Automations

Focus on repetitive tasks first:

  • Follow-up activity creation after meetings
  • Welcome emails for new leads
  • Deal assignment based on territory/product
  • Stage progression based on completed activities

Document Your Workflows

Keep a record of all automations:

  • Name each automation clearly
  • Note the trigger and all actions
  • Test each automation before activating
  • Review automations quarterly

Test Thoroughly

  1. Create a test deal with sample data
  2. Trigger the automation manually
  3. Verify all actions execute correctly
  4. Check that the right person receives notifications
  5. Confirm activities appear in the right place

Monitor and Adjust

  • Check automation logs weekly for errors
  • Ask sales team for feedback on automated processes
  • Adjust timing if activities are created too early/late
  • Remove automations that are no longer needed

Avoid Over-Automation

Don't automate everything:

  • Personal outreach should remain personal
  • Complex negotiations need human judgment
  • High-value deals deserve manual attention
  • Review automations that might seem impersonal

Common Mistakes

  • Too many triggers: Can cause duplicate activities
  • Missing conditions: Automations fire when they shouldn't
  • Wrong timing: Activities due too soon or too late
  • No testing: Errors go unnoticed until they affect real deals

Frequently Asked Questions

Can I create multiple actions in one automation?

Yes, Pipedrive allows you to add multiple actions to a single trigger. You can create activities, send emails, update fields, and more—all from one automation.

Do automations work on existing deals?

Automations trigger on events, not existing data. However, you can manually trigger automations by updating a deal field or moving it to a different stage.

Can I disable an automation temporarily?

Yes, go to Settings → Automation, find your automation, and toggle it off. You can re-enable it anytime without losing your configuration.

What happens if an automation fails?

Pipedrive logs automation failures in the activity feed. Check Settings → Automation → Logs to see error details. Most failures are due to missing required fields or permission issues.

Can I see which automation created an activity?

Yes, activities created by automations show "Created by automation" in the activity details. You can also see the automation name in the activity log.

Conclusion

Pipedrive automation saves hours of manual work while ensuring consistent sales processes. Start with basic deal stage movements and follow-up activities, then expand to more complex workflows as you get comfortable.

Recommended Implementation Order:

  2. Post-meeting follow-ups
  3. Won deal handoff to customer success
  4. Email sequences for lead nurturing
  5. Integration with Slack/Teams for notifications

Step 2: Set Up Real-Time Verification (15-30 minutes)

Integrate Proofy's API into your signup forms:

// Example: Real-time verification with Proofy API
const verifyEmail = async (email) => {
  const response = await fetch(
    `https://api.proofy.io/verify?email=${email}&apikey=${API_KEY}`
  );
  const result = await response.json();
  
  if (result.status === 'valid') {
    return { valid: true, message: 'Email verified successfully' };
  } else if (result.status === 'invalid') {
    return { valid: false, message: 'Invalid email address' };
  } else {
    return { valid: false, message: 'Unable to verify email' };
  }
};

Best Practices for Automation

✅ Do's

  • Verify at signup: Catch invalid emails before they enter your list
  • Use webhooks: Don't poll the API—let Proofy notify you when done
  • Implement rate limiting: Protect your API quota from spikes
  • Cache results: Don't re-verify emails you've recently checked
  • Log everything: Track verification attempts for debugging

❌ Don'ts

  • Don't verify on every send: Cache results for 30-90 days
  • Don't delete "risky" emails immediately: Test them first
  • Don't ignore webhook errors: Implement retry logic
  • Don't expose API keys: Always use server-side calls

Common Mistakes to Avoid

  1. Not implementing retry logic: API calls fail—handle gracefully
  2. Verifying too frequently: Email validity doesn't change daily
  3. Ignoring rate limits: Proofy has rate limits—respect them
  4. Not handling timeouts: Set appropriate timeouts for API calls
  5. Deleting without testing: Always test "risky" emails before removing

Troubleshooting Common Issues

Issue: API returning timeout errors

Solution: Implement exponential backoff retry logic. Start with a 1-second delay, then double for each retry. Limit retries to 3 attempts.

Issue: Webhook not receiving completion notifications

Solution: Verify your webhook URL is publicly accessible. Check firewall rules and ensure your server accepts POST requests. Add logging to confirm receipt.

Issue: Too many "unknown" results

Solution: Some domains have strict firewalls that prevent verification. For business emails, consider accepting "unknown" as potentially valid and monitoring engagement.

Frequently Asked Questions

How accurate is Proofy's email verification?

Proofy claims 98%+ accuracy for email verification. In our testing, we found accuracy rates between 95-99% depending on the email domain type. Business emails may have slightly lower verification rates due to corporate firewalls.

Can I use Proofy for real-time form validation?

Yes, Proofy's API is designed for real-time verification. However, always call the API from your server (not client-side JavaScript) to protect your API key. Typical response time is under 2 seconds.

What's the difference between "risky" and "unknown" status?

"Risky" typically means catch-all domains or disposable email providers—emails that might be valid but have higher bounce risk. "Unknown" means the verification couldn't be completed, often due to firewall restrictions.

How many API calls can I make per minute?

Rate limits depend on your plan. Most plans support 100-1000 requests per second. Check your dashboard for specific limits and consider batching for bulk operations.
